# Git Sync Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a GitHub-backed manual sync MVP that uploads and restores MarkBuddy business data without syncing Git credentials.

**Architecture:** Reuse the existing backup payload as the sync file format, add a small shared GitHub Contents API provider, and route sync operations through the MV3 service worker. The side panel owns form rendering and user confirmation; provider and sync logic stay DOM-free and testable with Node assertions.

**Tech Stack:** Chrome MV3 extension APIs, vanilla JavaScript UMD-style modules, GitHub Contents REST API, Node.js assertion tests.
