# 知识资产型 UI 与交互设计

## 背景

MarkBuddy 当前已经具备网页划线、摘录找回、唤醒重温、Markdown 导出、JSON 备份和 GitHub 仓库同步等能力。功能层面已经不只是一个网页高亮工具，而是一个本地优先的网页摘录资产库。

当前界面的主要问题不是功能缺失，而是信息架构仍偏向“功能陈列”：首页同时展示搜索、标签、排序、收藏卡片、摘录、提醒、导出和删除；设置页承担了偏好、Git 同步和备份；今日重温页已经有专注感，但上下文提示较弱。用户可以完成操作，但不容易一眼理解 MarkBuddy 的核心价值是“保存、找回、重温和带走自己的网页知识资产”。

本设计把 MarkBuddy 的 UI 和交互方向调整为“知识资产型”。目标是让用户感觉自己在管理一个持续增长的个人网页知识库，而不是在维护一组浏览器插件数据。

## 目标

- 让默认首页明确成为“摘录库”，优先呈现知识片段的内容价值。
- 将“今日重温”塑造成重新遇见重要摘录的专注流程，而不是任务清单。
- 将 Markdown 导出、JSON 备份和 GitHub 同步收束为“数据资产”心智。
- 降低设置页的压力，让设置只承载偏好，不承载主要数据工作流。
- 优化关键文案，使其围绕“摘录、重温、数据归属、长期保存”表达。
- 保持现有数据模型和核心功能不变，先做信息架构和交互表达升级。

## 非目标

- 不重写高亮保存、文本恢复、SM-2 排期、Git 同步或备份解析逻辑。
- 不引入账户系统、云服务、团队协作或社交高亮。
- 不把 MarkBuddy 改造成考试式复习软件。
- 不在本阶段新增 AI 摘要、语义搜索或第三方笔记软件直连。
- 不改变现有本地优先和用户自有 GitHub 仓库同步原则。

## 产品结构

MarkBuddy 主界面分为三个一级工作区：

```txt
摘录库
今日重温
数据资产
```

`摘录库` 是默认首页，回答“我保存了什么、为什么重要、如何找回”。

`今日重温` 是专注模式，回答“哪些摘录今天值得重新看一眼”。

`数据资产` 集中承载 Markdown 导出、JSON 备份、GitHub 同步和恢复，回答“我的知识如何带走、同步和长期保存”。

设置入口保留，但降级为偏好配置，例如主题、高亮颜色、域名分组、唤醒收藏开关和高级同步配置。Git 同步的日常操作不应主要藏在设置页里。

## 首页：摘录库

### 信息层级

首页卡片从“网页收藏条目”调整为“知识片段容器”。每张卡片按以下顺序展示：

```txt
标题
来源 · 保存时间 · 标签

摘录预览
备注预览

跳转原文 · 稍后重温/已加入重温/今天该看 · 导出 · 更多
```

标题和摘录预览是主信息。来源、时间、标签、划线数量和操作按钮降低视觉权重。

如果同一网页有多条摘录，卡片仍以网页为容器，但优先展示最有代表性的 1 到 3 条摘录。超过数量时显示“还有 N 条摘录”，点击后展开。

### 搜索与过滤

搜索框继续保留在顶部，但 placeholder 调整为：

```txt
搜索标题、来源、摘录或备注...
```

标签栏继续作为横向筛选，但不应喧宾夺主。筛选结果统计使用知识资产语义：

```txt
9 个来源 · 13 条摘录
```

而不是强调“网页/划线”的技术模型。

### 摘录操作

单条摘录的重温状态使用更明确的文案：

- 未加入：`稍后重温`
- 已加入：`已加入重温`
- 已到期：`今天该看`

取消入口可放在更多菜单或已加入状态的二级操作中，避免每条摘录默认出现红色“取消”按钮。

导出单个网页或单条摘录仍然保留，但默认弱化为图标或更多菜单操作。

### 空状态

空状态文案应表达知识库从第一条摘录开始：

```txt
还没有摘录
在网页上选中值得保留的内容，保存为你的第一条知识片段。
```

无搜索结果文案：

```txt
没有找到相关摘录
换个关键词，或清除筛选条件再试试。
```

## 今日重温

今日重温模式保持独立专注页。页面结构调整为：

```txt
今日重温        1 / 3        退出

来源：runoob.com
保存于：2 天前

摘录卡片
备注

跳转原文

不重要了    再提醒我    仍然有用
```

摘录卡片中优先展示摘录文本。如果有备注，备注在卡片内作为“我的笔记”展示。

`跳转原文` 不应远离摘录卡片。它应位于卡片内或卡片下方，让用户在判断价值时能快速回到上下文。

三档反馈继续映射现有 SM-2 逻辑，但文案调整为价值判断：

- `不重要了`：对应低评分或关闭后续提醒。
- `再提醒我`：对应中间评分，保留较近的再次提醒。
- `仍然有用`：对应高评分，延长下次提醒间隔。

页面完成后显示小结：

```txt
今日重温完成
仍然有用 N 条 · 再提醒我 N 条 · 不重要了 N 条
```

## 数据资产

新增或重组一个“数据资产”工作区，用来集中展示导出、同步和备份。

### Markdown 导出

导出区提供：

```txt
导出当前结果
导出全部摘录
复制 Markdown
```

导出说明使用场景化文案：

- 当前结果：适合把本次搜索或标签筛选后的材料带到笔记软件。
- 全部摘录：适合定期整理完整知识库。
- 复制 Markdown：适合临时粘贴到 Obsidian、Notion 或文档。

### GitHub 同步

GitHub 同步从配置表单升级为状态优先：

```txt
已连接 zky-git/markbuddy-data
上次同步：2026/7/8 01:08
Commit：be9ac72

立即同步
从仓库恢复
同步设置
```

当未配置时，展示引导卡：

```txt
把摘录同步到你自己的 GitHub 仓库
保留版本历史，不经过 MarkBuddy 服务器。

开始配置
```

Token、Owner、Repo、Branch 和文件路径放入“同步设置”或高级配置折叠区。日常用户不需要每次看到完整表单。

主要按钮文案调整：

- `上传到 Git` 改为 `同步到仓库`
- `从 Git 恢复` 改为 `从仓库恢复数据`
- `测试连接` 保留在高级配置中
- `清除配置` 放入危险操作区

### JSON 备份

JSON 备份保留为完整恢复能力：

```txt
导出完整备份
导入完整备份
```

说明文案：

```txt
完整备份用于恢复扩展数据；Markdown 适合进入笔记和写作流程。
```

## Markdown 输出格式

Markdown 导出应更像知识材料，而不是系统报表。推荐结构：

```markdown
## Python3 字典 | 菜鸟教程

- Source: https://www.runoob.com/python3/python3-dictionary.html
- Saved: 2026-07-07
- Tags: #python #基础语法

> 删除字典元素

笔记：
1. 字典删除单个元素 del dict[key]
2. 列表删除单个元素 del list[2]
3. 元组不能删除单个元素
```

多条摘录按同一页面聚合。单条摘录不再默认使用 `1. Highlight` 这类机械标题，除非用户没有标题或上下文。

## 文案原则

- 使用“摘录”替代大部分“划线”，面向用户价值而不是实现方式。
- 使用“来源”替代部分“网页”，强调知识出处。
- 使用“重温”替代“复习”，降低考试和任务压力。
- 使用“数据资产”“同步到仓库”“完整备份”表达数据归属。
- 危险操作需要明确后果，例如“从仓库恢复数据会覆盖本机摘录，但不会覆盖同步配置”。

## 架构影响

本阶段主要影响 UI 层：

- `side-panel/panel.html`：新增或重组一级工作区、数据资产入口和文案。
- `side-panel/panel.css`：调整卡片层级、导航、数据资产区和重温页视觉。
- `side-panel/panel.js`：调整渲染结构、工作区切换、重温按钮文案和 Markdown 入口。
- `side-panel/git-sync.js`：保留现有消息协议，调整 Git 同步在数据资产页的状态展示与操作入口。
- `side-panel/export-markdown.js`：调整 Markdown 输出结构，并支持复制 Markdown 时复用同一生成逻辑。

后台消息、同步引擎和存储模型不需要为本阶段改变。

## 测试

需要覆盖以下重点：

- 首页仍能渲染按域名分组和不分组两种模式。
- 搜索、标签筛选和排序在新“摘录库”结构下仍然生效。
- 单条摘录的重温状态文案正确显示为“稍后重温”“已加入重温”或“今天该看”。
- 今日重温页三档反馈仍正确调用现有回顾更新逻辑。
- 数据资产页的 Markdown 导出、复制、JSON 备份、Git 同步和从仓库恢复入口可用。
- Git 同步未配置、已配置、同步中、成功和失败状态均有清晰反馈。
- Markdown 输出符合新的知识材料结构，保留来源、保存时间、标签、摘录和备注。
- 设置页仍能修改主题、高亮颜色、域名分组和唤醒收藏开关。

## 推进顺序

建议分三步实施：

1. 重构首页摘录库卡片和核心文案。
2. 优化今日重温专注页和反馈文案。
3. 新增或重组数据资产工作区，并调整 Markdown 输出格式。

这三个阶段可以独立验收。第一阶段完成后，MarkBuddy 的“知识资产型”定位就会在默认首页中开始显现；第二阶段强化差异化；第三阶段把“数据属于用户”变成可见产品价值。

## 实施决策

- 一级工作区使用顶部紧凑分段入口，放在 header 下方。侧边栏高度有限，底部导航会与今日重温反馈按钮和滚动内容抢空间。
- 今日重温里的“不重要了”第一版只映射为低评分并推迟提醒，不直接关闭提醒。用户主动取消重温仍通过摘录库中的状态操作完成，避免一次反馈导致内容永久退出队列。
- Markdown 复制功能纳入第一版数据资产页。它复用 Markdown 导出生成逻辑，只增加剪贴板写入和成功/失败提示。
