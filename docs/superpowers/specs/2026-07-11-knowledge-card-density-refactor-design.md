# 知识卡片去重与层级重构设计

## 目标

解决网页卡片在展开后同时显示摘要与完整内容，造成摘录、备注和操作重复的问题。重构后，同一条内容在任一状态下只出现一次。

## 状态设计

### 收起状态

- 显示标题、来源、保存时间和标签。
- 最多显示 2 条摘录摘要。
- 摘录摘要只保留左侧高亮色标和单行文本，不使用大面积底色。
- 备注不在摘要区重复展示；有备注时仅显示轻量“有备注”标记。
- 底部显示“展开 N 条摘录”和导出操作。

### 展开状态

- 隐藏全部摘要内容。
- 只显示完整摘录列表、完整备注和编辑操作。
- 展开按钮改为“收起摘录”，箭头同步反转。
- 再次收起时恢复最多 2 条摘要。

## 视觉层级

- 卡片只保留一个外层容器，不在摘要区创建嵌套卡片。
- 标题是第一视觉层级；来源、时间和标签为弱化元数据。
- 摘录正文使用透明背景和左侧颜色线。
- 备注使用浅色文字区和细边界，不与摘录争夺主视觉。
- 删除、复制、编辑等操作仅在展开后的对应摘录中出现。

## 交互规则

- 展开状态继续按网页 URL 保存在当前侧边栏会话中。
- 搜索命中摘录或备注时自动展开该网页卡片，并隐藏摘要。
- “还有 N 条摘录”与“展开 N 条摘录”触发同一展开行为。
- 无摘录时不提供展开动作。

## 实现边界

- 不修改收藏、高亮、备注、重温和同步的数据模型。
- 不改变删除、编辑、复制、导出和跳转原文的业务逻辑。
- 改动集中在 `side-panel/panel.js`、`side-panel/panel.css` 与结构测试。

## 验证标准

- 展开时摘要列表必须隐藏，完整列表必须显示。
- 收起时完整列表必须隐藏，摘要列表必须显示。
- 展开按钮文案在“展开 N 条摘录”和“收起摘录”之间切换。
- 摘录摘要不再输出完整备注文本。
- 现有侧边栏、重温、备份、导出和 Git 同步测试全部通过。
