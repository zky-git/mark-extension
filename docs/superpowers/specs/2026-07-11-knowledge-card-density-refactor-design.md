# 知识卡片去重与层级重构设计

## 目标

删除网页卡片中重复的“摘录摘要区”。每条摘录及其备注只在统一摘录列表中出现一次，不再提供展开或收起两套展示状态。

## 卡片结构

每个网页卡片固定使用以下顺序：

1. 标题、来源、保存时间和删除网页操作。
2. 网页标签和添加标签操作。
3. 统一摘录列表。
4. 网页级 Markdown 导出操作。

不再渲染独立摘要区、“查看 N 条摘录”、“还有 N 条摘录”或展开箭头。

## 摘录列表

- 默认直接展示该网页下的全部摘录。
- 每条摘录只包含左侧高亮色标、摘录正文、重温状态和删除操作。
- 有备注时在摘录下方显示完整备注，并保留复制、编辑和删除操作。
- 无备注时显示弱化的“添加备注”入口。
- 摘录正文和备注不在卡片其他位置再次出现。

## 视觉层级

- 网页卡片是唯一外层容器，不再使用摘要卡片嵌套。
- 摘录之间使用细分隔线和留白区分，不使用大面积浅色背景。
- 摘录正文为主要内容，备注使用轻量背景或细边线作为次级内容。
- 标签、来源、时间、重温状态和操作按钮均弱化，避免与正文争夺注意力。
- 网页级导出操作放在卡片底部并保持低强调。

## 交互规则

- 不存在展开、收起或摘要切换状态。
- 搜索只过滤和高亮真实摘录列表，不触发额外的卡片状态变化。
- 删除、编辑、复制、重温、导出和跳转原文的业务逻辑保持不变。
- 无摘录的网页仍显示网页信息和导出操作，但不显示空的摘录容器。

## 实现边界

- 删除 `highlight-preview-list` 及其相关渲染和样式。
- 删除 `expandedBookmarkUrls`、`card-expand-btn` 和 `highlights-list.open` 状态逻辑。
- 完整摘录列表改为始终可见。
- 不修改收藏、高亮、备注、重温和同步的数据模型。
- 改动集中在 `side-panel/panel.js`、`side-panel/panel.css` 与结构测试。

## 验证标准

- 每条摘录正文在单个网页卡片中只渲染一次。
- 每条备注在单个网页卡片中只渲染一次。
- 卡片中不存在摘要列表、展开按钮或收起状态。
- 完整摘录列表默认可见。
- 现有侧边栏、重温、备份、导出和 Git 同步测试全部通过。
