# 待重温数量并入 Tab 设计

## 目标

移除摘录库中的独立待重温提示，将待重温数量直接显示在顶部“今日重温”标签后。

## 界面行为

- 移除 `#review-banner` 及其中的图标、数量文案和“看看这些”按钮。
- “今日重温”标签在任意状态下显示为 `今日重温（N）`，其中 `N` 是当前待重温条目数；例如 `今日重温（3）`。
- 点击该标签仍会进入既有的重温流程。

## 数据与同步

- 继续使用既有 `GET_DUE_REVIEWS` 查询待重温队列。
- 初始化时加载数量；重温状态更新、退出重温和完成重温后重新同步标签文字。
- 功能关闭时数量显示为 `0`，不显示额外提示。

## 测试

- 结构测试断言独立横幅不存在。
- 结构测试断言今日重温标签包含可更新的数量节点，并由脚本刷新。

## 范围

仅调整侧边栏的数量呈现位置；不改变重温队列、SM-2 排期、徽标或“今日重温”入口行为。
