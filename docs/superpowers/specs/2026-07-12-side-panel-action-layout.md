# 侧边面板操作布局收尾说明

## 目标

整理侧边面板中导出、重温和删除等高频操作的位置，使顶部导航更轻，卡片操作更贴近对象本身。

## 当前界面约定

- 顶部工作区只显示 `摘录库`、`今日重温（N）` 和 `数据资产` 三个入口。
- `今日重温（N）` 中的 `N` 来自待重温队列数量，包含 `0`。
- 顶部 tab 行不再展示全局下载按钮。
- 全局 Markdown 导出、复制 Markdown、JSON 备份和 GitHub 同步集中在 `数据资产` 工作区。
- 单个网页的 Markdown 下载按钮显示在该网页卡片的标签行右侧。
- 单条划线的“稍后重温 / 已加入重温 / 今天该看”按钮位于划线行右侧，并与文本垂直居中。
- 单条划线删除按钮常显为灰色关闭图标，hover 时才显示红色危险状态。

## 交互范围

本次收尾只调整呈现位置与视觉状态，不改变以下行为：

- Markdown 导出数据格式与导出逻辑。
- JSON 备份/恢复逻辑。
- GitHub 同步逻辑。
- SM-2 排期与待重温队列。
- 删除收藏、删除划线和更新划线重温状态的消息协议。

## 测试覆盖

`tests/panel-structure.test.js` 覆盖以下结构约束：

- 顶部 tab 行不渲染 `#export-toggle-btn`。
- 工作区标签仍保持三个等宽 tab。
- 单卡下载按钮渲染在标签行右侧，而不是卡片底部 footer。
- 划线行内容与操作垂直居中。
- 划线删除按钮默认可见、默认灰色、hover 进入红色危险状态。
